Formularios en Angular

Los formularios son una parte esencial de muchas aplicaciones web, permitiendo a los usuarios ingresar y enviar datos. Angular ofrece dos enfoques principales para manejar formularios: **Template-Driven** y **Reactive Forms**.

Tipos de Formularios en Angular

  • Template-Driven Forms: Basados en plantillas, ideales para formularios simples y menos estructurados.
  • Reactive Forms: Basados en código, ofrecen mayor control y flexibilidad, adecuados para formularios complejos.

Formularios Template-Driven

Características

  • Fácil de usar y comprender.
  • Utiliza directivas de Angular como ngModel.
  • Ideal para formularios simples.

Ejemplo de Formulario Template-Driven

Vamos a crear un formulario para agregar un nuevo post utilizando el enfoque Template-Driven.

Actualizar hola-mundo.component.ts

        
nuevoPost: any = { title: '', body: '' };

crearNuevoPost(): void {
  if (this.nuevoPost.title && this.nuevoPost.body) {
    this.apiService.crearPost(this.nuevoPost).subscribe(
      data => {
        this.posts.unshift(data);
        this.nuevoPost = { title: '', body: '' };
      },
      error => console.error(error)
    );
  }
}

Actualizar hola-mundo.component.html

        <h3>Crear Nuevo Post (Template-Driven)</h3>
<form #postForm="ngForm" (ngSubmit)="crearNuevoPost()">
  <div>
    <label for="title">Título:</label>
    <input type="text" id="title" name="title" [(ngModel)]="nuevoPost.title" required />
  </div>
  <div>
    <label for="body">Contenido:</label>
    <textarea id="body" name="body" [(ngModel)]="nuevoPost.body" required></textarea>
  </div>
  <button type="submit" [disabled]="!postForm.form.valid">Crear Post</button>
</form>

En este formulario:

  • Se utilizan las directivas ngModel para enlazar los campos del formulario con las propiedades del componente.
  • El atributo required asegura que los campos sean obligatorios.
  • El botón de envío está deshabilitado si el formulario no es válido.

Validaciones en Formularios Template-Driven

Angular proporciona varias validaciones integradas que puedes utilizar para asegurar la integridad de los datos ingresados por el usuario.

Ejemplo de Validaciones

        <form #postForm="ngForm" (ngSubmit)="crearNuevoPost()">
  <div>
    <label for="title">Título:</label>
    <input type="text" id="title" name="title" [(ngModel)]="nuevoPost.title" required minlength="5" />
    <div *ngIf="postForm.controls.title?.invalid && (postForm.controls.title?.dirty || postForm.controls.title?.touched)">
      <small>El título es obligatorio y debe tener al menos 5 caracteres.</small>
    </div>
  </div>
  <div>
    <label for="body">Contenido:</label>
    <textarea id="body" name="body" [(ngModel)]="nuevoPost.body" required></textarea>
    <div *ngIf="postForm.controls.body?.invalid && (postForm.controls.body?.dirty || postForm.controls.body?.touched)">
      <small>El contenido es obligatorio.</small>
    </div>
  </div>
  <button type="submit" [disabled]="!postForm.form.valid">Crear Post</button>
</form>

Este ejemplo agrega validaciones de longitud mínima y mensajes de error que se muestran cuando los campos no cumplen con los requisitos.

Formularios Reactivos

Características

  • Más control y flexibilidad sobre el formulario.
  • Gestión programática de formularios mediante código.
  • Mejor para formularios complejos con lógica avanzada.

Configuración de Formularios Reactivos

Para utilizar formularios reactivos, primero debes importar el módulo ReactiveFormsModule en app.module.ts:

        import { ReactiveFormsModule } from '@angular/forms';

@NgModule({
  imports: [
    BrowserModule,
    FormsModule,
    ReactiveFormsModule,
    RouterModule.forRoot(routes),
    HttpClientModule
  ],
  // ... otros códigos
})
export class AppModule { }

Creando un Formulario Reactivo

Vamos a crear un formulario reactivo para actualizar un post existente.

Modificar hola-mundo.component.ts

        import { Component, OnInit } from '@angular/core';
import { DataService } from '../data.service';
import { ApiService } from '../api.service';
import { FormGroup, FormControl, Validators } from '@angular/forms';

@Component({
  selector: 'app-hola-mundo',
  templateUrl: './hola-mundo.component.html',
  styleUrls: ['./hola-mundo.component.css']
})
export class HolaMundoComponent implements OnInit {
  datos: string[] = [];
  posts: any[] = [];
  nuevoPost: any = { title: '', body: '' };
  formularioActualizar: FormGroup;

  constructor(private dataService: DataService, private apiService: ApiService) { 
    this.formularioActualizar = new FormGroup({
      title: new FormControl('', [Validators.required, Validators.minLength(5)]),
      body: new FormControl('', Validators.required)
    });
  }

  ngOnInit(): void {
    this.datos = this.dataService.obtenerDatos();
    this.apiService.obtenerPosts().subscribe(
      data => this.posts = data.slice(0, 5),
      error => console.error(error)
    );
  }

  mensaje: string = "¡Bienvenido al binding de datos!";

  cambiarMensaje(nuevoMensaje: string): void {
    this.mensaje = nuevoMensaje;
  }

  nombre: string = "Angular";

  enviarNotificacion(): void {
    // Implementación previa
  }

  crearNuevoPost(): void {
    if (this.nuevoPost.title && this.nuevoPost.body) {
      this.apiService.crearPost(this.nuevoPost).subscribe(
        data => {
          this.posts.unshift(data);
          this.nuevoPost = { title: '', body: '' };
        },
        error => console.error(error)
      );
    }
  }

  actualizarPost(post: any): void {
    this.formularioActualizar.setValue({
      title: post.title,
      body: post.body
    });
    // Lógica para enviar la actualización
  }
}

Prueba del Formulario Reactivo

Implementa la lógica para manejar la actualización en hola-mundo.component.ts:

        actualizarPostSubmit(): void {
  if (this.formularioActualizar.valid) {
    const updatedPost = this.formularioActualizar.value;
    // Supongamos que tienes un método para actualizar el post en el servicio
    this.apiService.actualizarPost(updatedPost).subscribe(
      data => {
        // Actualiza la lista de posts con los nuevos datos
        const index = this.posts.findIndex(p => p.id === data.id);
        if (index !== -1) {
          this.posts[index] = data;
        }
        this.formularioActualizar.reset();
      },
      error => console.error(error)
    );
  }
}

Este método verifica la validez del formulario y luego envía los datos actualizados al servicio para su procesamiento.

Práctica Recomendada

Elige el enfoque de formulario que mejor se adapte a la complejidad de tu aplicación. Para formularios simples, los Template-Driven son suficientes, mientras que para formularios más complejos y con lógica avanzada, los Reactive Forms ofrecen mayor control.
