Comunicación entre Componentes

En una aplicación Angular, es común que diferentes componentes necesiten comunicarse entre sí. Angular proporciona varias formas de manejar esta comunicación de manera eficiente y mantenible.

Tipos de Comunicación

  • Input y Output: Comunicación entre componentes padre e hijo.
  • Servicios: Comunicación entre componentes no relacionados mediante servicios compartidos.
  • Observables y Subjects: Manejo de eventos y datos reactivos entre componentes.

Comunicación Padre-Hijo con @Input y @Output

1. Usando @Input

La decoración @Input permite que un componente padre pase datos a un componente hijo.

Ejemplo de @Input

En app.component.ts, define una propiedad para pasar al hijo:

mensajePadre: string = "Mensaje desde el padre";

En la plantilla app.component.html, pasa la propiedad al componente hijo:

        <app-hola-mundo [mensaje]="mensajePadre"></app-hola-mundo>

En hola-mundo.component.ts, importa Input y define la propiedad receptora:

        import { Component, OnInit, Input } from '@angular/core';

@Component({
  selector: 'app-hola-mundo',
  templateUrl: './hola-mundo.component.html',
  styleUrls: ['./hola-mundo.component.css']
})
export class HolaMundoComponent implements OnInit {
  @Input() mensaje: string = "";

  constructor() { }

  ngOnInit(): void {
  }

  // ... otras propiedades y métodos
}

En hola-mundo.component.html, muestra el mensaje recibido:

        <h2>{{ mensaje }}</h2>

Al ejecutar la aplicación, verás el mensaje enviado desde el componente padre.

2. Usando @Output

La decoración @Output permite que un componente hijo emita eventos al componente padre.

Ejemplo de @Output

En hola-mundo.component.ts, importa Output y EventEmitter, y define un evento:

        import { Component, OnInit, Input, Output, EventEmitter } from '@angular/core';

@Component({
  selector: 'app-hola-mundo',
  templateUrl: './hola-mundo.component.html',
  styleUrls: ['./hola-mundo.component.css']
})
export class HolaMundoComponent implements OnInit {
  @Input() mensaje: string = "";
  @Output() notificar: EventEmitter<string> = new EventEmitter<string>();

  constructor() { }

  ngOnInit(): void {
  }

  enviarNotificacion(): void {
    this.notificar.emit('¡Notificación desde el hijo!');
  }

  // ... otras propiedades y métodos
}

En la plantilla hola-mundo.component.html, añade un botón para emitir el evento:

        <button (click)="enviarNotificacion()">Enviar Notificación</button>

En el componente padre app.component.html, maneja el evento:

        <app-hola-mundo [mensaje]="mensajePadre" (notificar)="manejarNotificacion($event)"></app-hola-mundo>

En app.component.ts, define el método para manejar el evento:

        manejadorNotificacion(evento: string): void {
  alert(evento);
}

Ahora, al hacer clic en el botón del componente hijo, se disparará una alerta con el mensaje de notificación.

Comunicación entre Componentes No Relacionados

Cuando los componentes no tienen una relación padre-hijo, la comunicación se puede manejar mediante servicios compartidos.

Usando un Servicio Compartido

Ya hemos creado el servicio DataService que puede actuar como intermediario para compartir datos entre componentes.

Ejemplo de Comunicación con Servicios

Supongamos que tienes dos componentes: ComponenteA y ComponenteB.

En data.service.ts, añade un Subject para emitir eventos:

        import { Injectable } from '@angular/core';
import { Subject } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class DataService {
  private mensajeSubject = new Subject<string>();
  mensaje$ = this.mensajeSubject.asObservable();

  enviarMensaje(mensaje: string): void {
    this.mensajeSubject.next(mensaje);
  }

  obtenerDatos(): string[] {
    return ["Dato 1", "Dato 2", "Dato 3"];
  }
}

En componente-a.component.ts, inyecta el servicio y emite un mensaje:

        import { Component } from '@angular/core';
import { DataService } from '../data.service';

@Component({
  selector: 'app-componente-a',
  templateUrl: './componente-a.component.html',
  styleUrls: ['./componente-a.component.css']
})
export class ComponenteAComponent {

  constructor(private dataService: DataService) { }

  enviarMensaje(): void {
    this.dataService.enviarMensaje('Mensaje desde Componente A');
  }
}

En la plantilla componente-a.component.html, añade un botón para enviar el mensaje:

<button (click)="enviarMensaje()">Enviar Mensaje a Componente B</button>

En componente-b.component.ts, suscríbete al mensaje del servicio:

        import { Component, OnInit } from '@angular/core';
import { DataService } from '../data.service';

@Component({
  selector: 'app-componente-b',
  templateUrl: './componente-b.component.html',
  styleUrls: ['./componente-b.component.css']
})
export class ComponenteBComponent implements OnInit {
  mensajeRecibido: string = '';

  constructor(private dataService: DataService) { }

  ngOnInit(): void {
    this.dataService.mensaje$.subscribe(mensaje => {
      this.mensajeRecibido = mensaje;
    });
  }
}

En la plantilla componente-b.component.html, muestra el mensaje recibido:

        <p>Mensaje Recibido: {{ mensajeRecibido }}</p>

Con esta configuración, al hacer clic en el botón del ComponenteA, el ComponenteB recibirá y mostrará el mensaje.

Uso de Observables y Subjects

Los Observables y Subjects de RxJS son herramientas poderosas para manejar flujos de datos asíncronos y eventos en Angular.

¿Qué es un Observable?

Un Observable es una colección de futuros valores o eventos. Puede ser observado por múltiples suscriptores, permitiendo la reacción a cada nuevo valor emitido.

¿Qué es un Subject?

Un Subject es tanto un Observable como un Observer. Puede emitir nuevos valores y ser suscrito para recibir estos valores, facilitando la comunicación entre componentes.

Práctica Recomendada

Utiliza servicios compartidos con Observables y Subjects para manejar la comunicación entre componentes no relacionados. Esto promueve una arquitectura más limpia y desacoplada.
