Servicios y Dependencias en Angular

Los servicios en Angular son clases que proporcionan funcionalidades específicas que pueden ser compartidas entre múltiples componentes. Utilizan el patrón de diseño de Inyección de Dependencias para gestionar las dependencias y facilitar la reutilización del código.

¿Qué es un Servicio?

Un servicio es una clase con una responsabilidad bien definida, como la gestión de datos, la comunicación con APIs, la autenticación de usuarios, etc. Los servicios ayudan a mantener los componentes ligeros y enfocados en la presentación y la lógica de la interfaz de usuario.

Creando un Servicio

Para generar un nuevo servicio llamado data, utiliza Angular CLI:

ng generate service data

Este comando creará dos archivos:

  • data.service.ts: Contiene la lógica del servicio.
  • data.service.spec.ts: Archivo de pruebas unitarias.

Implementando el Servicio

En data.service.ts, vamos a implementar un método para obtener datos:

        import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class DataService {

  constructor() { }

  obtenerDatos(): string[] {
    return ["Dato 1", "Dato 2", "Dato 3"];
  }
}

El decorador @Injectable indica que esta clase puede tener dependencias inyectadas en ella. La propiedad providedIn: 'root' asegura que el servicio se registre a nivel de aplicación, haciéndolo disponible para todos los componentes.

Inyectando el Servicio en un Componente

Para utilizar el servicio DataService en un componente, debemos inyectarlo en el constructor del componente.

Modificar hola-mundo.component.ts

        import { Component, OnInit } from '@angular/core';
import { DataService } from '../data.service';

@Component({
  selector: 'app-hola-mundo',
  templateUrl: './hola-mundo.component.html',
  styleUrls: ['./hola-mundo.component.css']
})
export class HolaMundoComponent implements OnInit {
  datos: string[] = [];

  constructor(private dataService: DataService) { }

  ngOnInit(): void {
    this.datos = this.dataService.obtenerDatos();
  }

  mensaje: string = "¡Bienvenido al binding de datos!";

  cambiarMensaje(nuevoMensaje: string): void {
    this.mensaje = nuevoMensaje;
  }

  nombre: string = "Angular";
}

Aquí, el servicio DataService se inyecta en el componente mediante el constructor. Luego, en el método ngOnInit, llamamos a obtenerDatos() para obtener los datos y asignarlos a la propiedad datos.

Actualizar la Plantilla hola-mundo.component.html

        <h2>{{ mensaje }}</h2>
<button (click)="cambiarMensaje('Has actualizado el mensaje!')">Cambiar Mensaje</button>

<input [(ngModel)]="nombre" placeholder="Escribe tu nombre"/>
<p>Hola, {{ nombre }}!</p>

<h3>Datos del Servicio:</h3>
<ul>
  <li *ngFor="let dato of datos">{{ dato }}</li>
</ul>

Esto renderizará una lista de datos obtenidos desde el servicio.

Inyección de Dependencias en Angular

La inyección de dependencias es un patrón de diseño que permite que las clases reciban sus dependencias de fuentes externas en lugar de crearlas internamente. En Angular, esto se gestiona mediante el sistema de proveedores y el inyector.

Proveedores

Los proveedores son responsables de crear instancias de servicios. Al especificar providedIn: 'root', Angular crea una única instancia del servicio que se comparte en toda la aplicación.

Inyectores

El inyector es el mecanismo que crea y entrega las instancias de los servicios a los componentes o a otros servicios que los requieran.

Beneficios de Usar Servicios

  • Reutilización de Código: Facilita la reutilización de lógica común entre múltiples componentes.
  • Mantenimiento: Centraliza la lógica de negocio, haciendo que sea más fácil de mantener y actualizar.
  • Pruebas Unitarias: Facilita la creación de pruebas unitarias al separar la lógica de negocio de la lógica de presentación.
  • Modularidad: Promueve una arquitectura más modular y organizada.
